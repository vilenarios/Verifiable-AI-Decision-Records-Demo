How verification works
Show the canonical bytes ↔ signed commitment click to expand
The proof has two parts. Ar.io anchors a tiny signed commitment — just hashes and a signature, no source data. MLflow stores the canonical bytes that produced the hash. Anyone with both can verify they match.

Verifier recomputes:
SHA-256(canonical bytes) = payload_hash {% if v and v.hash_match is sameas true %} ✓ matches {% elif v and v.hash_match is sameas false %} ✗ doesn't match {% endif %}
What you can't do. Change either side without breaking the equality. The signature on the signed commitment binds payload_hash to the signer's key.
What this isn't. Canonical bytes contain hashes of model artifacts, not the model weights themselves — the model file stays in MLflow, only its hash is anchored.
