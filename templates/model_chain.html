How verification works
Show the canonical bytes ↔ signed commitment for each chain link click to expand
The proof has two parts. Ar.io anchors a tiny signed commitment — just hashes and a signature, no source data. MLflow stores the canonical bytes that produced the hash. Anyone with both can verify they match. Each node in the chain follows the same pattern; the example below shows the training run link.

What you can't do. Change either side without breaking the equality. The signature on the signed commitment binds payload_hash to the signer's key — and each event's commitment also includes previous_hash, chaining the links so a tamper at any point breaks every event after it.
What this isn't. Canonical bytes contain hashes of model artifacts, not the model weights themselves — the model file stays in MLflow, only its hash is anchored.
