Verify this independently

Don't take this report's verdict on faith — confirm it yourself. The easy path uses the open-source ario-mlflow plugin; the forensic path uses only standard cryptographic tools so an auditor can verify without installing anything from this project.

Easy mode · plugin CLI 
pip install ario-mlflow
ario-mlflow verify trace {{ decision_id }}

Fetches the signed envelope from ar.io, pulls the canonical bytes from MLflow, and prints the same four-check verdict shown above. One command, no file shuffling.

Verify without trusting the plugin (advanced)

For auditors whose threat model includes "the plugin itself could lie." Uses only curl, shasum, Python's nacl, and jcs (RFC 8785 JSON canonicalisation).

{% if envelope.arweave_tx_id %} 1 · Fetch the signed envelope from ar.io 
curl -sS https://arweave.net/{{ envelope.arweave_tx_id }} -o envelope.json
python3 -c "import json; print(json.load(open('envelope.json'))['payload_hash'])"
2 · Fetch the canonical bytes from MLflow 
python3 - <<'PY'
import mlflow
mlflow.set_tracking_uri("<your MLflow tracking URI>")
path = mlflow.artifacts.download_artifacts(
    run_id="{{ rec.mlflow_run_id or '' }}",
    artifact_path="ario/predictions/{{ decision_id }}/payload.json",
    dst_path="."
)
print("saved →", path)
PY
3 · Re-hash the canonical bytes 
shasum -a 256 payload.json
# Compare the first column to the hash printed in step 1.
# If they match, the canonical payload hasn't been altered since anchoring.
4 · Verify the cryptographic signature 
pip install pynacl jcs

python3 <<'PY'
import json, binascii, jcs
from nacl.signing import VerifyKey
env = json.load(open("envelope.json"))
sig = binascii.unhexlify(env.pop("signature"))
pk = binascii.unhexlify(env["public_key"])
VerifyKey(pk).verify(jcs.canonicalize(env), sig)
print("Signature: VALID")
PY

The signature covers the JCS-canonicalised envelope minus the signature field itself. Any byte change since signing makes verify() raise nacl.exceptions.BadSignatureError.
